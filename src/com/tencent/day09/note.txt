*****一、super关键字
	super关键字代表当前类的父类对象，通过super关键字能访问父类中的方法与属性
	
	super.父类方法()
	super.父类属性
	super()		调用父类中的构造方法
	
*****一个类被继承以后，子类进行构造时，不管子类是否有参还是无参
	  父类默认调用的是无参构造
	  
二、final关键字用于修饰类、属性、方法

	final修饰的类不能被继承
	
	final修饰的变量称为常量，值不能被改变
	一般常量结合static使用，称为静态常量,此对象永远不会被回收
	
	final修饰的方法不能被重写

*****三、抽象类
	1、什么是抽象类
		使用abstract修饰的类称为抽象类，抽象类不能使用new关键字创建对象，
		只能与子类进行构造
		
	2、抽象类的编写格式
	public abstract class 类名{
		//类体
		
	}
	
	3、抽象方法
		使用abstract修饰的方法，称为抽象方法，编写格式:
			访问修饰符	abstract 返回值类型	方法名(参数列表)
			public abstract void sayLanguage();
			
		特点:
			a、抽象方法只能定义在抽象类或接口中
			b、抽象方法只有定义，没有实现(没有方法体)
			c、子类继承父类，一定要重写父类中的抽象方法，
				如果子类也是一个抽象类，则不需要重写
			d、抽象方法不能使用private修饰
			
	4、上转型对象
		父类与子类进行构造，创建的对象称为上转型对象，编写格式如下:
			父类		上转型对象名	=new 子类();
			
		上转型对象访问范围:
			a、父类中非私有的属性和方法
			b、子类重写父类的方法
			
		子类对象>上转型对象>父类对象
		
		上转型对象属于对象多态，同种类型创建对象，调用同一个方法，出现不同的内容
		
	 5、子类对象与父类之间的转换
	  子类转父类		隐式转换		大转小
	  父类转子类		强制转换		小转大
	 	
		
*****四、接口
	1、什么是接口
		使用Interface关键字来定义一个接口，接口与抽象类很类似，编写如下:
			public interface 接口名{
				//接口体		
			}
			
			接口名与文件名必须一样，首字母大写
	2、接口的特点
		a、接口中只能定义属性与方法，
		定义的属性全是公共静态常量
		定义的方法全是公共抽象方法()
		
		b、接口中不能有构造方法、普通方法、静态方法、final修饰的方法
		
		c、子类可以实现多个接口，接口可以继承多个接口，接口不能继承类
		子类必须重写父类接口中的所有方法
		
	3、接口回调
		父接口与子类进行构造创建对象，称为接口回调
		父接口	接口回调对象名=new 子类();
		
		接口回调访问范围:
			a、父接口中的属性与方法
			b、子类重写父接口的方法
		
	4、java中的继承关系
		类与类是单继承			单继承
		接口与接口是多继承		多继承
		类与接口是多实现		多实现
		接口与类				接口不能继承或实现类
